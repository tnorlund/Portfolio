## What the review validated
- Re-OCR pathway is directionally correct; atlas retained only for logos/bitmaps.

  (0.94/0.97/0.84, no cross-region mislabels).

## What it corrected (honest)
- **My "0 mislabels / pipeline solid" was overstated.** The scorer counts a label as correct by region-
  membership, so garbage satisfies it: `'='` token gets `B-TAX` (Amazon) while SUBTOTAL is absent; register
  prefix `'B1'` absorbs `TIME` (Costco); `GRAND_TOTAL` lands on a value-corrupted `$10.13`. type_recall is
  gameable (one good-enough token = "recovered"; saturated 0.00 fields hide regressions).
- **The root cause is the RENDERER, not propagation.** All 3 fail the image gate (realism 2‚Äì3.5/10): the
  totals/payment block is a tiny faint low-contrast micro-font ‚Üí Apple Vision drops SUBTOTAL (Amazon),
  GRAND_TOTAL (Sprouts), and silently corrupts values (Costco 12.99‚Üí12.00, Amazon $18.13‚Üí$10.13,
  MasterCard‚Üíasterar). Plus decimal-dots-as-dashes, broken logos, missing QR/barcode, center-justified items,
  and synthetic totals that don't even reconcile (17.40+0.65 ‚âÝ 18.13).

## Blind spots named
- Corrupted OCR values poison downstream consumers ‚Üí store synthetic GT value SEPARATELY from the OCR token.
- No pre-OCR arithmetic sanity check (subtotal+tax==total; no empty value fields) ‚Äî a free deterministic reject.
- Sample size 3, no statistics ‚Äî need a 50‚Äì100 batch harness before any conclusion.
- Apple Vision / OS version drift threatens reproducibility of the baked-in noise ‚Äî pin + log the Vision version.
- Global CER hides per-field noise shape (monetary fields are the wrong shape) ‚Äî calibrate per field/section.
- Alignment robustness untested on OCR split/merge across lines, dense totals, repeated identical amounts.

## Prioritized actions (from the review)
1. **Quarantine the 3 renders** ‚Äî do not propagate/train. (unanimous)
2. **Deterministic raw-OCR coverage gate** after Apple Vision, before propagation: every required GT region
   must have nearby raw OCR coverage, else quarantine. This REPLACES opus image-review as the recall gate
   (automatable at scale).
3. **Harden `reocr_score.py`**: add label precision, false-positives on GT-`O` regions, per-entity confusion,
   per-field value/CER accuracy. Reject 1-char/punctuation value labels unless geometry strongly supports.
4. **Fix the renderer at source**: uniform full-ink full-size mono digits across totals/payment; fix decimal-dot
   glyph; repair logos; render QR + barcode + all line items; pre-OCR reject for non-reconciling/empty totals.
5. Per-token propagation diagnostics in every artifact (label_source, matched GT, alignment score, coverage, conf).
6. Keep synthetic GT value separate from OCR token; define value-recovery/normalization path.
7. 50‚Äì100 batch through the harness; recall+precision+coverage per field; calibrate per-field noise vs real.
8. Only then wire behind a flag and A/B synthetic-pretrain ‚Üí real-finetune vs clean-token, on a held-out REAL set.
